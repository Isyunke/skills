# /resume-import — 简历导入

导入你的现有简历，解析内容，考察你是否真正参与过这些项目。

## 为什么需要考察？

你提交的简历可能是：
- ✅ 你真实参与的项目
- ⚠️ 你参与但贡献很少的项目（马赛克项目）
- ❌ 虚构或 AI 生成的项目

**考察的目的不是质疑你，而是：**
1. 确保你能回答面试官的细节提问
2. 发现你的知识缺陷
3. 针对性准备面试

## Overview

```
[用户：导入简历]
  ↓
[Phase 0: 读取简历文件]
  ↓
[Phase 1: 解析简历内容]
  ↓
[Phase 2: 生成初步 Profile]
  ↓
[Phase 3: 考察用户经历]
  ↓
[Phase 4: 分类处理]
  ├─ 有效经历 → 存入 project.md
  ├─ 马赛克项目 → 存入学习/面试指导
  └─ 无效经历 → 询问用户后处理
  ↓
[Phase 5: 更新技能树]
  ↓
[Phase 6: 询问模板策略] ⭐ 新增
  ├─ ① 保留原始模板
  │    ├─ HTML/MD → 提取模板骨架 → 存入 templates/imported/
  │    └─ PDF → 降级为优化建议模式
  └─ ② 使用系统/自定义模板 → 交给 resume-build
  ↓
[Phase 7: 输出结果]
```

## Constants

- **SUPPORTED_FORMATS = ["pdf", "doc", "docx", "md", "html"]** — 支持的简历格式
- **CONFIDENCE_THRESHOLD = 0.7** — 信心阈值，低于此值需要考察

## Inputs

| 必填 | 来源 |
|---|---|
| `<resume-path>` | 用户提供的简历文件路径 |
| `.resume-state.json` | 项目根 |

## Workflow

### Phase 0: 读取简历文件

1. 检查文件是否存在
2. 检查文件格式是否支持
3. 根据格式调用对应的解析器

**支持的格式**：

| 格式 | 解析方式 | 说明 |
|---|---|---|
| `.md` | 直接读取 | Markdown 格式 |
| `.html` | 直接读取 | HTML 格式 |
| `.txt` | 直接读取 | 纯文本格式 |
| `.pdf` | 调用 `tools/resume_parser.py` | 需要安装依赖 |
| `.doc` | 调用 `tools/resume_parser.py` | 需要安装依赖 |
| `.docx` | 调用 `tools/resume_parser.py` | 需要安装依赖 |

- 联系方式：XX

|---|---|---|---|
| XX 公司 | 后端工程师 | 2022-2024 | ... |

### Phase 3: 考察用户经历

**考察策略**：

#### 3.1 项目深挖（必问）

对每个项目，用 STAR 法则考察：

```
AI：聊聊 XX 项目？
   这个项目背景是什么？（Situation）
用户：...
AI：你具体负责什么？（Task）
用户：...
AI：你用了什么技术？为什么选这个技术？
用户：...
AI：遇到什么困难？怎么解决的？
用户：...
AI：结果如何？有数据吗？
用户：...
```

#### 3.2 技术细节考察

对每个技能，考察技术细节：

```
AI：你说你精通 Python，
   能聊聊 Python 的 GIL 是什么吗？
用户：...
AI：Python 的装饰器是什么？怎么实现？
用户：...
```

#### 3.3 场景问题

针对项目场景，问具体问题：

```
AI：如果让你重新设计这个系统，
   你会怎么改进？
用户：...
AI：如果并发量增加 10 倍，
   你会怎么优化？
用户：...
```

#### 3.4 信心评估

根据用户回答，评估信心：

| 信心 | 标准 | 处理 |
|---|---|---|
| 🟢 高 (≥0.7) | 回答详细、有深度、有具体案例 | 存入 project.md |
| 🟡 中 (0.4-0.7) | 回答基本正确，但缺少细节 | 标记为"马赛克项目" |
| 🔴 低 (<0.4) | 回答模糊、错误、或回避问题 | 标记为"需学习" |

### Phase 4: 分类处理

#### 4.1 有效经历（🟢 高信心）

**处理**：
1. 存入 `profile/projects/proj-<NNN>-<short>.md`
2. 更新 `profile/self-profile.md` 的技能树
3. 标记为"已验证"

**project.md 结构**：
```markdown
# [项目名称]

**项目 ID**: proj-001
**时间**: 2024.01 - 2024.06
**角色**: 后端开发
**验证状态**: ✅ 已验证
**验证时间**: 2026-06-13

## 项目背景
（从用户回答中提取）

## 我的职责
（从用户回答中提取）

## 技术方案
- 语言：Python
- 框架：Django

## 具体工作
（从用户回答中提取）

## 项目成果
（从用户回答中提取，量化！）

## 技能标签
| 技能 | 熟练度 | 证据强度 |
|---|---|---|
| Python | 精通 | 🟢 强 |

## 面试要点
- 最大挑战：...
- 解决方案：...
```

#### 4.2 马赛克项目（🟡 中信心）

**定义**：用户参与过，但贡献很少，无法回答细节问题。

**处理**：
1. 存入 `learning/mosaic-projects/mosaic-<NNN>-<short>.md`
2. 生成面试指导（针对可能被问到的问题）
3. 生成学习计划（补充知识缺陷）

**mosaic-<NNN>-<short>.md 结构**：
```markdown
# 马赛克项目：[项目名称]

**项目 ID**: mosaic-001
**时间**: 2024.01 - 2024.06
**角色**: 参与者（非核心）
**验证状态**: ⚠️ 马赛克项目

## 项目背景
（从简历中提取）

## 你的参与程度
- 参与时间：X 个月
- 主要工作：...
- 核心贡献：有限

## 知识缺陷

### 技术层面
- [ ] 不了解系统架构细节
- [ ] 不了解核心技术选型原因
- [ ] 不了解性能优化方案

### 业务层面
- [ ] 不了解业务背景
- [ ] 不了解核心业务流程
- [ ] 不了解业务难点

## 面试指导

### 可能被问到的问题

**Q: 介绍一下这个项目**
**建议回答**：
"这个项目是 XX，我在其中负责 XX 模块..."
（诚实说明你的参与程度，不要夸大）

**Q: 你在这个项目中遇到的最大困难？**
**建议回答**：
"我主要负责 XX 部分，遇到的困难是..."
（诚实说明你的真实困难，不要编造）

**Q: 这个项目的架构是什么？**
**建议回答**：
"我主要负责 XX 模块，整体架构是..."
（诚实说明你了解的部分，不要假装了解全部）

### 回答策略

1. **诚实为本**：不要夸大你的贡献
2. **聚焦自己**：重点描述你负责的部分
3. **承认不足**：如果不了解，诚实说"这部分不是我负责的"
4. **展示学习**：如果你学到了什么，诚实分享

## 学习计划

### 目标
- 补充项目相关的技术知识
- 准备面试可能问到的问题

### 学习内容
1. [ ] 了解系统整体架构
2. [ ] 了解核心技术选型
3. [ ] 了解性能优化方案
4. [ ] 了解业务背景和流程

### 学习资源
- 项目文档（如有）
- 技术博客
- 源码阅读
```

#### 4.3 无效经历（🔴 低信心）

### Phase 6: 询问模板策略 ⭐ 新增

在内容解析和经历考察全部完成后，询问用户如何处理简历模板。

**为什么放在最后？**
- 考察过程中可能发现马赛克项目需要裁剪
- 用户看到最终解析结果后，才能理性决定是否保留原模板
- 模板策略不影响内容解析和经历考察的流程

**询问话术**：

```
✅ 内容解析和经历考察已完成。

接下来，你希望如何处理简历的排版模板？

a) 保留原始模板 — 我会尽量复用你原简历的排版风格
b) 使用系统模板 — 从官方模板中选择
c) 使用自定义模板 — 你提供一个 HTML 模板文件
```

#### 选项 A：保留原始模板

根据源文件格式，采用不同策略：

**A1. 源文件是 HTML**

```
检测到你的简历是 HTML 格式，可以直接提取模板骨架。

正在分析模板结构...
- 布局类型：双栏
- 主色调：#2c3e50
- 字体：Microsoft YaHei
- Section 结构：头部 → 自我评价 → 技能 → 工作经历 → 项目经历 → 教育背景

✅ 模板骨架已提取，保存至 templates/imported/my-resume-template.html

后续生成简历时，可以使用这个模板。
```

**处理流程**：
1. 读取 HTML 文件
2. 分析 CSS 结构（布局、颜色、字体、间距）
3. 提取 section 骨架（哪些 section、什么顺序）
4. 将内容区域替换为模板变量（`{{name}}`、`{{skills}}` 等）
5. 保存为 `templates/imported/<name>.html`
6. 在 `.resume-state.json` 中记录 `imported_template` 字段

**A2. 源文件是 Markdown**

```
检测到你的简历是 Markdown 格式，可以提取结构模板。

正在分析 Markdown 结构...
- 标题层级：# 姓名 → ## Section → ### 条目
- 内容结构：头部 → 技能 → 工作经历 → 项目经历 → 教育背景
- 列表风格：无序列表，用 - 开头

✅ 结构模板已提取，保存至 templates/imported/my-resume-template.md
```

**处理流程**：
1. 读取 Markdown 文件
2. 分析标题层级和 section 结构
3. 提取内容骨架
4. 保存为 `templates/imported/<name>.md`

**A3. 源文件是 PDF** ⚠️ 降级处理

```
PDF 文件无法直接提取模板结构（PDF 是"画"出来的，不是"编"出来的）。

我会为你生成一份优化建议文档，你可以手动在原文件上修改。

📁 优化建议：resumes/imported/optimize-suggestions.md
📁 原始简历：profile/imported-resume.pdf（已备份）

你可以：
a) 查看优化建议，手动修改原文件
b) 改用系统模板重新生成
```

**处理流程**：
1. 不提取模板（技术上不可行）
2. 将原 PDF 备份到 `profile/imported-resume.pdf`
3. 生成 `resumes/imported/optimize-suggestions.md`，包含：
   - 关键词优化建议（JD 匹配度分析）
   - 量化数据补充建议
   - STAR 法则改写建议
   - 排版优化建议（基于文本分析推断）
4. 用户手动在原 PDF 上修改

**optimize-suggestions.md 结构**：

```markdown
# 简历优化建议

## 基于 JD：jd-001-xx公司后端

### 关键词优化
- ❌ 缺少：微服务、分布式、Redis
- ⚠️ 弱化：Python（建议补充具体项目）

### 量化优化
- 当前："负责性能优化"
- 建议："负责核心系统性能优化，响应时间从 500ms 降至 150ms，提升 70%"

### STAR 改写
- 当前："参与电商系统开发"
- 建议：
  - Situation: 电商平台日活 100w+，高峰期响应超时
  - Task: 负责后端性能优化模块
  - Action: 优化 SQL 查询 + 引入 Redis 缓存层
  - Result: P99 延迟从 800ms 降至 120ms

### 排版建议
- 建议控制在 1-2 页
- 技能清单放在显眼位置
- 项目经历按时间倒序排列
```

## Key Rules

1. **诚实为本**——不要帮助用户虚构经历
2. **考察为主**——目的是发现知识缺陷，不是质疑用户
3. **分类处理**——有效/马赛克/无效，分别处理
4. **学习导向**——所有缺陷都转化为学习机会
5. **模板自主**——用户决定是否保留原始模板，系统不强制
6. **PDF 降级**——PDF 无法提取模板时，诚实告知并提供替代方案

- 「PDF 一定能提取模板」 → 拒绝。PDF 是"画"出来的，技术上无法可靠逆向
