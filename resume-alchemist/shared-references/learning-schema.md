## 学习路径

### 阶段 1：理论基础（Day 1-3）

**目标**：理解分布式系统的基本概念

**内容**：
1. CAP 理论
   - 一致性 (Consistency)
   - 可用性 (Availability)
   - 分区容错 (Partition tolerance)
   - 三者只能满足其二

2. 一致性协议
   - Paxos
   - Raft
   - ZAB

3. 分布式存储
   - 分布式文件系统
   - 分布式数据库

**验证**：
- [ ] 能解释 CAP 理论
- [ ] 能解释 Raft 协议
- [ ] 能画出分布式存储架构图

---

### 阶段 2：实践项目（Day 4-7）

**目标**：通过实践加深理解

**项目**：
1. 搭建简单的分布式 KV
   - 使用 Go/Python 实现
   - 支持基本的 GET/PUT 操作
   - 实现简单的 Raft 共识

2. 实现分布式锁
   - 基于 Redis 实现
   - 理解分布式锁的挑战

**验证**：
- [ ] 完成分布式 KV 项目
- [ ] 完成分布式锁项目
- [ ] 能解释实现细节

---

### 阶段 3：深入理解（Day 8-10）

**目标**：深入理解分布式系统的挑战

**内容**：
1. 分布式事务
   - 2PC
   - 3PC
   - Saga

2. 微服务架构
   - 服务拆分
   - 服务治理
   - 服务监控

**验证**：
- [ ] 能解释分布式事务
- [ ] 能设计微服务架构

---

### 阶段 4：项目整合（Day 11-14）

**目标**：将学习成果应用到实际项目

**内容**：
1. 重构现有项目
   - 应用分布式系统知识
   - 优化系统架构

2. 准备面试
   - 整理学习笔记
   - 准备面试回答

**验证**：
- [ ] 完成项目重构
- [ ] 准备好面试回答

## 与简历的关系

learning/ 是简历的**补充**：

| 学习 | 简历 |
|---|---|
| 学习技能 | 技能清单更新 |
| 实践项目 | 项目经历补充 |
| 学习笔记 | 面试准备 |

**原则**：学习是为了补充简历的缺口，不能脱离简历学习。
