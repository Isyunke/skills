# 项目文档 Schema (Project Schema)

被这些子 skill 引用：`resume-mine`、`resume-profile`、`resume-build`。

---

## 文件位置

```
<user-resume-project>/profile/projects/proj-<NNN>-<short>.md
```

- `<NNN>`：三位数字，从 001 开始
- `<short>`：3-8 字中文或英文短名

---

## 完整 schema

```markdown
# [项目名称]

**项目 ID**: proj-001
**时间**: 2024.01 - 2024.06
**角色**: 后端开发
**团队**: 5人
**公司**: XX 公司（可选）
**验证方式**: 代码溯源 / 对话深挖 / 简历导入
**验证状态**: ✅ 已验证 / ⏳ 待验证
**验证得分**: 92/100（可选，仅代码溯源验证时生成）

## 项目背景

（Situation - 项目背景是什么？为什么做这个项目？）

## 我的职责

（Task - 你的任务是什么？负责什么模块？）

## 技术方案

- 语言：Python
- 框架：Django
- 数据库：MySQL, Redis
- 工具：Docker, Git
- 架构：微服务 / 单体 / ...

## 具体工作

（Action - 你做了什么？怎么做的？）

1. 负责 XX 模块开发
2. 优化 XX 性能
3. ...

## 项目成果

（Result - 结果如何？量化！）

- 性能提升 50%
- 用户量增长 100%
- 节省人力 XX 人天
- ...

## 技能标签

| 技能 | 熟练度 | 证据强度 |
|---|---|---|
| Python | 精通 | 🟢 强 |
| Django | 熟练 | 🟢 强 |
| MySQL | 熟练 | 🟡 弱 |

## 面试要点

- **最大挑战**：...
- **解决方案**：...
- **学到的**：...
- **如果重来**：...

## 相关链接

- GitHub: https://github.com/...
- 文档: https://...
```

---

## 字段说明

### 元数据

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `项目 ID` | string | ✅ | proj-<NNN>，唯一标识 |
| `时间` | string | ✅ | YYYY.MM - YYYY.MM |
| `角色` | string | ✅ | 你在项目中的角色 |
| `团队` | string | ❌ | 团队人数 |
| `公司` | string | ❌ | 所属公司 |

### STAR 段落

| 段落 | 对应 | 必填 | 说明 |
|---|---|---|---|
| 项目背景 | Situation | ✅ | 项目背景、业务场景 |
| 我的职责 | Task | ✅ | 你的任务、负责范围 |
| 技术方案 | - | ✅ | 技术栈、架构 |
| 具体工作 | Action | ✅ | 你做了什么、怎么做的 |
| 项目成果 | Result | ✅ | 结果、量化数据 |

### 技能标签

| 字段 | 类型 | 说明 |
|---|---|---|
| 技能 | string | 技能名称 |
| 熟练度 | enum | "了解" / "熟练" / "精通" |
| 证据强度 | emoji | 🔴 无 / 🟡 弱 / 🟢 强 |

### 面试要点

| 字段 | 说明 |
|---|---|
| 最大挑战 | 面试官必问 |
| 解决方案 | STAR 的 Action |
| 学到的 | 成长反思 |
| 如果重来 | 改进空间 |

## 创建时机

- `/resume-mine` 引导用户描述项目时创建
- 用户主动说"更新项目"时更新
- `/resume-jd` 分析 JD 时，如果发现新项目需要补充，提示用户
- `/resume-verify` 代码溯源验证通过后创建（从 project-tmd.md 重命名）

---

## 更新规则

- **不覆盖历史版本**：每次更新 append 到文件末尾，或创建新版本
- **原子写**：使用 .tmp → rename
- **git 版本控制**：用户可回滚到任意版本

---

## 与简历的关系

project.md 是简历内容的**数据源**：

| project.md | 简历 |
|---|---|
| 项目背景 | 项目描述（精简版） |
| 我的职责 | 项目角色 |
| 技术方案 | 技术栈 |
| 具体工作 | 工作内容（STAR 精简版） |
| 项目成果 | 量化成果 |
| 技能标签 | 技能列表 |

**原则**：简历内容必须来自 project.md，不能凭空编写。
