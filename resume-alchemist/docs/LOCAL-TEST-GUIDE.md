A: 我分析了慢查询日志，优化了 SQL，引入了 Redis 多级缓存，还加了消息队列做异步
R: 响应时间从 500ms 降到 150ms，并发从 300 QPS 提到 1200 QPS

## 五、重点关注清单

以下是**必须重点关注**的测试点，按风险排序：

### 🔴 高风险（必须验证）

1. **模板渲染**：生成的 HTML 简历是否能正确显示？CSS 是否生效？
2. **PDF 导出**：weasyprint/playwright 在你的环境能否正常安装和运行？
3. **简历导入的考察逻辑**：马赛克项目判定是否合理？用户感受如何？
4. **数据一致性**：多次操作后 state.json 是否与实际文件一致？

### 🟡 中风险（应该验证）

5. **STAR 深挖的用户体验**：追问是否让人感到被审问？
6. **JD 匹配度的准确性**：同一个 JD 分析两次，结果是否一致？
7. **原子写**：是否有残留的 .tmp 文件？

### 🟢 低风险（有空验证）

8. **学习计划的实用性**：推荐的资源是否具体可执行？
9. **面试准备的针对性**：问题是否与简历和 JD 匹配？
