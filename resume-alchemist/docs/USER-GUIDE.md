## 快速开始

### 第一步：初始化

在你的求职项目目录中，打开 Claude Code 并说：

```
初始化简历炼金术士
```

或

```
/init
```

系统会问你 4 个问题：
1. 你想从事什么职位？
2. 你目标什么行业？
3. 你有多少年经验？
4. 你有什么现成的简历或项目文档吗？

### 第二步：深挖经历

```
聊聊我的经历
```

系统会用 STAR 法则引导你挖掘真实经历。

### 第三步：分析 JD

```
分析这个 JD [粘贴 JD 文本]
```

系统会自动解析 JD，生成岗位分析。

### 第四步：生成简历

```
针对这个 JD 优化简历
```

系统会根据你的经历和 JD，生成针对性简历。

### 第五步：导出 PDF

```
导出 PDF
```

系统会将 HTML 简历转换为 PDF。

## 详细流程

│       ↑                                              │

### 阶段 1：深挖经历

**目标**：挖掘你的真实经历，建立项目档案

**触发词**：
- "聊聊我的经历"
- "深挖一下"
- "更新项目"
- "补充经历"

**流程**：
1. 系统读取你的技能树
2. 用 STAR 法则引导你描述项目
3. 提取关键信息
4. 生成项目文档
5. 更新技能树

**STAR 法则**：
- **S**ituation：项目背景是什么？
- **T**ask：你的任务是什么？
- **A**ction：你做了什么？
- **R**esult：结果如何？（量化！）

### 阶段 2：分析 JD

**目标**：解析 JD，找出技能匹配度

**触发词**：
- "分析这个 JD"
- "看看这个岗位"
- "分析岗位"

**流程**：
1. 系统读取 JD 文本
2. 提取关键信息
3. 匹配技能树
4. 生成岗位分析
5. 创建岗位文档

**输出**：
- 岗位概览
- 核心要求（匹配度、证据、缺口）
- 优先级排序
- 优化建议
- 学习计划

### 阶段 3：定制简历

**目标**：根据 JD 生成针对性简历

**触发词**：
- "针对这个 JD 优化简历"
- "生成简历"
- "写简历"

**流程**：
1. 系统读取 JD 分析
2. 读取技能树和项目文档
3. 匹配经历与 JD
4. 生成简历内容
5. 应用模板
6. 输出 HTML

**输出**：
- 针对性简历（HTML）
- 历史版本（resume-v<N>.html）

### 阶段 4：面试准备

**目标**：生成面试题库和回答策略

**触发词**：
- "准备面试"
- "模拟面试"
- "面试指导"

**流程**：
1. 系统读取简历和 JD 分析
2. 生成自我介绍
3. 生成项目深挖问题
4. 生成技术问题
5. 生成行为问题
6. 生成反问环节

**输出**：
- 自我介绍（1分钟版）
- 项目深挖问题
- 技术问题
- 行为问题
- 反问环节

### 阶段 5：学习补充

**目标**：分析技能缺口，制定学习计划

**触发词**：
- "这个技能我不会"
- "学什么"
- "技能缺口"
- "学习计划"

**流程**：
1. 系统读取 JD 分析
2. 识别技能缺口
3. 制定学习计划
4. 推荐学习资源
5. 创建学习文档

**输出**：
- 学习路径
- 学习资源
- 实践项目
- 验证方式

## 子 Skill 详解

### resume-init（初始化）

**职责**：引导用户完成初始化，建立项目结构

**触发词**：
- "初始化"
- "init"
- "首次使用"
- "setup"

**流程**：
1. 检测当前状态
2. 询问 4 个问题
3. 创建项目结构
4. 生成初始模板
5. 给出下一步建议

### resume-mine（深挖经历）

**职责**：通过对话式引导，挖掘用户的项目和工作经历

**触发词**：
- "聊聊我的经历"
- "深挖一下"
- "更新项目"
- "补充经历"

**流程**：
1. 读取现有档案
2. 引导式提问（STAR 法则）
3. 提取关键信息
4. 生成/更新 project.md
5. 更新 self-profile.md

### resume-import（简历导入）⭐ 新增

**职责**：导入现有简历，解析内容，考察用户经历真实性

**触发词**：
- "导入简历"
- "我有简历"
- "解析简历"
- "从简历导入"

**支持格式**：
- `.md` - Markdown 格式
- `.html` - HTML 格式
- `.txt` - 纯文本格式
- `.pdf` - PDF 格式（需要 pdfplumber）
- `.docx` - Word 文档（需要 python-docx）
- `.doc` - 旧版 Word（需要 antiword）

**流程**：
1. 读取简历文件
2. 解析简历内容
3. 生成初步 Profile
4. 考察用户经历（STAR 法则）
5. 分类处理：
   - 有效经历 → 存入 project.md
   - 马赛克项目 → 存入学习/面试指导
   - 无效经历 → 询问用户后处理
6. 更新技能树

**新概念**：

**马赛克项目**：
- 定义：用户参与过但贡献很少的项目
- 特征：无法回答技术细节、无法说明架构设计
- 处理：存入 `learning/mosaic-projects/`，生成面试指导和学习计划

**知识缺口**：
- 定义：用户无法验证的技能或项目
- 特征：无法回答基本问题、缺少技术细节
- 处理：存入 `learning/knowledge-gaps/`，生成学习计划

**使用示例**：

```
用户：导入简历
AI：好的，请提供你的简历文件路径
用户：./my-resume.pdf
AI：✅ 简历解析完成
    📊 解析结果：
    - 工作经历：2 段
    - 项目经历：3 个
    - 技能清单：8 项

    现在开始考察你的经历...

    请聊聊 XX 项目的背景是什么？
用户：...
```

## 常见问题

### Q: 这个工具和 ChatGPT 有什么区别？

A: ChatGPT 是通用助手，会告诉你"怎么写简历"。
简历炼金术士是你的专属求职顾问，会帮你：
1. 深挖你的真实经历
2. 针对每个 JD 定制简历
3. 准备面试
4. 发现技能缺口

### Q: 我没有项目经历怎么办？

A: 通过 `/resume-mine` 引导你挖掘：
- 课程项目
- 个人项目
- 实习经历
- 工作经历

### Q: 简历内容会造假吗？

A: 不会。简历炼金术士有三条不可妥协原则：
1. 真实性原则：只记录真实经历
2. 针对性原则：针对 JD 优化
3. 证据链原则：每项技能必须有项目支撑

### Q: 如何保证简历质量？

A: 简历炼金术士有盲评估机制：
- 独立评估简历质量
- 检查关键词覆盖率
- 检查量化数据充分度
- 检查 STAR 法则完整度

### Q: 一份简历可以投多个 JD 吗？

A: 不建议。简历炼金术士的原则是"针对性原则"：
- 每份简历必须针对特定 JD 优化
- 针对性简历的面试通过率是通用简历的 3-5 倍
- 系统会强制提示"请为这个 JD 定制简历"

### Q: 如何更新技能树？

A: 通过以下方式更新：
1. 深挖经历后自动更新
2. 学习新技能后手动更新
3. 通过 `/resume-profile` 查看和更新

### Q: 如何管理多个 JD？

A: 每个 JD 都会创建独立的目录：
```
resumes/
├── jd-001-xx公司后端/
├── jd-002-yy公司全栈/
└── jd-003-zz公司算法/
```

### Q: 如何回滚简历版本？

A: 简历有版本管理：
- 当前版本：resume.html
- 历史版本：resume-v1.html, resume-v2.html, ...
- 可以通过 git 回滚

## 最佳实践

### 1. 先深挖经历，再分析 JD

```
✅ 正确流程：
聊聊我的经历 → 分析这个 JD → 针对这个 JD 优化简历

❌ 错误流程：
分析这个 JD → 针对这个 JD 优化简历（没有经历支撑）
```

### 2. 用 STAR 法则描述经历

```
✅ 好的描述：
Situation: 电商系统日活 100w+，性能下降
Task: 负责性能优化，目标响应时间降到 200ms
Action: 优化数据库查询，引入缓存
Result: 响应时间从 500ms 降到 150ms，提升 70%

❌ 差的描述：
我做了性能优化，性能提升了很多
```

### 3. 量化成果

```
✅ 好的量化：
- 性能提升 70%
- 用户量增长 100%
- 响应时间从 500ms 降到 150ms

❌ 差的量化：
- 性能提升了很多
- 用户量增长了不少
- 响应时间快了很多
```

### 4. 针对 JD 优化简历

```
✅ 好的做法：
- 分析 JD 要求
- 突出相关经历
- 使用 JD 关键词

❌ 差的做法：
- 一份简历投所有
- 不分析 JD
- 不使用关键词
```

### 5. 准备面试

```
✅ 好的做法：
- 准备自我介绍
- 准备项目深挖
- 准备技术问题
- 准备行为问题

❌ 差的做法：
- 不准备
- 只准备技术问题
- 不准备项目深挖
```

## 技巧

### 技巧 1：批量深挖

一次可以深挖多个项目：

```
用户：聊聊我做过的项目
AI：好的，先从最近的开始？
用户：好的，第一个项目是...
AI：...
用户：第二个项目是...
AI：...
```

### 技巧 2：更新项目

可以随时更新已有项目：

```
用户：更新项目 proj-001
AI：好的，你想更新什么？
用户：我想补充一些成果...
AI：...
```

### 技巧 3：对比 JD

可以分析多个 JD，选择最匹配的：

```
用户：分析这个 JD
AI：✅ JD 分析完成
用户：再分析这个 JD
AI：✅ JD 分析完成
用户：哪个更匹配？
AI：...
```

### 技巧 4：迭代优化

可以多次优化简历：

```
用户：优化一下简历
AI：✅ 简历已优化
用户：再优化一下
AI：✅ 简历已优化
```

### 技巧 5：学习补充

发现技能缺口后，可以制定学习计划：

```
用户：这个技能我不会
AI：✅ 学习计划已生成
用户：学完了，更新技能树
AI：✅ 技能树已更新
```
